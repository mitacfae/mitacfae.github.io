--- layout: post title: "How to Remotely Upgrade R2520G6 Firmware Using Redfish Commands" date: 2026-06-11 00:00:00 tags: [R2520G6, Redfish, Firmware, BIOS, BMC, CPLD, ROT] last_updated: 2026-06-11 --- ## Description [cite_start]When performing server maintenance, manual or onsite firmware updates often introduce numerous challenges and inefficiencies[cite: 2]. [cite_start]To improve operational efficiency and implement robust remote management, an automated solution is required to streamline the firmware update process for the R2520G6 server[cite: 2]. ## Solution [cite_start]This solution provides a comprehensive guide on how to utilize **Redfish API commands** to remotely upgrade the **BIOS, BMC, CPLD, and ROT** firmware of the R2520G6 server[cite: 2, 10]. ### Prerequisites Before executing any Redfish update commands, ensure that you complete the following preparatory steps: 1. [cite_start]**Power Off the Server**: Ensure that the R2520G6 is completely powered off before executing any Redfish upgrade commands[cite: 11, 12]. 2. [cite_start]**Prepare Firmware Files**: Upload the required out-of-band (OOB) firmware upgrade image files to your Linux working directory[cite: 9, 10, 13]. --- ## Detailed Firmware Upgrade Steps ### 1. BIOS Firmware Upgrade Process * [cite_start]**Step 1: Check Current Version** Verify the currently active BIOS version using the following Redfish command[cite: 20]: ```bash curl -u username:password -k -s https://{BMC_IP}/redfish/v1/UpdateService/FirmwareInventory/bios_active ``` [cite_start][cite: 21, 22] * [cite_start]**Step 2: Define Update Target** Use the `PATCH` command to define the update target as `bios_active`[cite: 23]: ```bash curl -k -u username:password -X PATCH https://{BMC_IP}/redfish/v1/UpdateService -H "Content-Type: application/json" -d '{"HttpPushUriOptions": {"HttpPushUriApplyTime": {"ApplyTime": "Immediate"}},"HttpPushUriTargets": ["bios_active"],"HttpPushUriTargetsBusy": true,"Oem": {"ApplyOptions": {"ClearConfig": true}}}' -w "\nHTTP_CODE=%{response_code}\n" ``` [cite: 24] *(Expected Response HTTP Code: `200`)* [cite: 25] * **Step 3: Execute Firmware Upgrade** Use the `POST` command to upload the corresponding upgrade image file[cite: 26]: ```bash curl -k -u username:password -X POST https://{BMC_IP}/redfish/v1/UpdateService/update -H "Content-Type: application/octet-stream" --data-binary "@{Image_path}" -w "\nHTTP_CODE=%{response_code}\n" ``` [cite: 27] *(Expected Response HTTP Code: `202`)* [cite: 28] * **Step 4: Wait and Verify** A new task will appear in the BMC Task menu[cite: 29]. **Please wait approximately 10 minutes** for the BIOS upgrade process to complete[cite: 29]. Once finished, check the firmware version again to verify success[cite: 30]. --- ### 2.
