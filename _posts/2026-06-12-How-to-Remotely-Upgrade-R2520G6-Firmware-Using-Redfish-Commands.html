How to Remotely Upgrade R2520G6 Firmware Using Redfish Commands

Posted on June 11, 2026

Description

When performing server maintenance, manual or onsite firmware updates often introduce numerous operational challenges and inefficiencies. To improve maintenance efficiency and enable robust remote management capability, this guide demonstrates how to remotely upgrade firmware through Redfish API commands on the MiTAC R2520G6 platform.

Solution Overview

This article provides step-by-step instructions to remotely upgrade the following firmware components:

  • BIOS
  • BMC
  • CPLD
  • ROT

All firmware updates are executed through Redfish API interfaces.

Prerequisites

  1. Ensure the R2520G6 server is powered off before starting firmware upgrades.
  2. Prepare the required Out-of-Band (OOB) firmware image files on your Linux host system.

1. BIOS Firmware Upgrade

Step 1 — Check Current BIOS Version

curl -u username:password -k -s \
https://{BMC_IP}/redfish/v1/UpdateService/FirmwareInventory/bios_active

Step 2 — Configure Upgrade Target

curl -k -u username:password \
-X PATCH https://{BMC_IP}/redfish/v1/UpdateService \
-H "Content-Type: application/json" \
-d '{
    "HttpPushUriOptions": {
        "HttpPushUriApplyTime": {
            "ApplyTime": "Immediate"
        }
    },
    "HttpPushUriTargets": [
        "bios_active"
    ],
    "HttpPushUriTargetsBusy": true,
    "Oem": {
        "ApplyOptions": {
            "ClearConfig": true
        }
    }
}' \
-w "\nHTTP_CODE=%{response_code}\n"

Expected HTTP Response: 200

Step 3 — Upload BIOS Firmware Image

curl -k -u username:password \
-X POST https://{BMC_IP}/redfish/v1/UpdateService/update \
-H "Content-Type: application/octet-stream" \
--data-binary "@{Image_path}" \
-w "\nHTTP_CODE=%{response_code}\n"

Expected HTTP Response: 202

Step 4 — Wait for Completion

A new firmware update task will appear in the BMC task menu. Please wait approximately 10 minutes before checking the BIOS version again.
